## Design Context

### Users
CEO of a health tech consultancy evaluating a software developer candidate's case study submission. This person sees dozens of dashboards and demos — they instantly recognize generic AI output. They value: clarity of thinking made visible, genuine craft, and evidence that the candidate can ship production-quality software. The demo will be screen-recorded via Loom and screenshots used in a PowerPoint presentation.

### Brand Personality
**Institutional precision** — like a Bloomberg terminal met a Swiss clinic. Not startup-playful, not corporate-boring. The feeling of a system built by someone who understands both the medicine and the engineering. Quiet confidence. Data speaks, chrome doesn't.

Three words: **Authoritative. Precise. Restrained.**

### Aesthetic Direction
- **Theme**: Dark — this is a monitoring/operations tool used by health campaign managers, likely in office settings with mixed lighting. Dark reduces eye strain during long analytical sessions.
- **NOT the AI palette**: No cyan-on-dark, no purple gradients, no glowing neon accents. These are the #1 tell of AI-generated dashboards.
- **References**: Linear (interaction quality, typography), Stripe Dashboard (data presentation clarity), Raycast (dark theme done right without being "gamer"), Bloomberg Terminal (information density that doesn't feel cluttered)
- **Anti-references**: Generic Tailwind admin templates, shadcn/ui default dark theme, anything with gradient text or glassmorphism
- **Cultural note**: Saudi health authority context — the design should feel institutional and trustworthy, not trendy. Arabic text appears in message previews (RTL) but the dashboard UI itself is English.

### Design Principles
1. **Data over decoration** — every pixel serves the analysis. No ornamental elements that don't convey information.
2. **Hierarchy through typography, not color** — use font weight, size, and spacing to create hierarchy. Color is reserved for status and data encoding only.
3. **Earned density** — information-dense where it matters (analytics, scoring breakdowns), spacious where the user needs to think (framework selection, message review).
4. **One memorable element per screen** — each screen has ONE thing that makes someone pause and look closer. Not everything is special; that's what makes the special things work.
5. **Motion with purpose** — animations exist to show state changes and data relationships, not to "look cool." Page transitions, number counters, and chart reveals — not bouncing icons.
